Snap, Flatpak and Appimage: a resource usage comparison.

Moltkex

I was curious about how snaps, flatpaks and appimages compare to each other in terms of resource usage, so I created a VM in QEMU, installed Debian Buster and made this simple comparison. Though I used Debian, I think this could be done in any other distro and results won't be that different, if any different at all.

The comparison process:

  • Installed Ksnip's Snap and Flatpak package and downloaded the appimage version as well.
  • Checked disk space before and after installing Snapd.
  • Checked disk space before and after installing Flatpak.
  • Checked Snap, Flatpak and Appimage package RAM and CPU usage with htop.
  • Checked Snap, Flatpak and Appimage package execution time by launching the app with time tool.

 Why Ksnip?

Ksnip is a small package with almost no dependencies but a little few, plus is the one I use to take screenshots, all the screenshots in this post were taken with it, besides, it's a wonderful app which does what it does and does it really well.  By the way, it is available in Buster via backports, though not the latest which you can get from its GitHub page.

I won't go into detail on how to install Snapd and Flatpak, there's plenty of info on the web about that, nor will I explain how to get or run Appimages, there's plenty of info on that too.

Disk Space 

Before installing Snapd, disk space available was 5.4GB, after installing it decreased by 1.8GB, so now it was 3.6GB.
