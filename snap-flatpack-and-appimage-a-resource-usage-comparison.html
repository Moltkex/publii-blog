Snap, Flatpack and Appimage: a resource usage comparison.

Moltkex

This a simple comparison on how Snaps, Flatpacks and Appimages use resources in a Debian Buster VM created with QEMU. While I used Debian, I think this could be done in any other distro and results won't be that different, if any different at all.

                                                       The comparison process:

  • Installed Ksnip's Snap and Flatpack package and download the appimage.
  • Checked disk space before and after installing Snapd.
  • Checked disk space before and after installing Flatpack.
  • Checked Snap, Flatpack and Appimage package RAM and CPU usage with htop.

 Why Ksnip?

Well, it's a small package with almost no dependencies but a little few, plus is the one I use to take screenshots, all the screenshots in this post were taken with it, besides, it's a wonderful app which does what it does really well.  By the way, it is available in Buster via backports, though not the latest which you can get from its GitHub page.

I won't go into detail on how to install Snapd and/or Flatpack, there's plenty of info on the web about that, nor will I explain how to get or run Appimages, there's plenty of info on that too. 

Before installing Snap and Flatpack disk space available was 5.4GB, after installing both it decreased by 1.8GB, so now it was 3.6GB, with snap using around 600mb and flatpack using as much as 1.2GB! The appimage is only a 27mb download!  

The first image is the flatpack pkg, the second one is the snap's and the third one is the appimage. As can be seen, the appimage uses slightly less resources than its counterparts.
